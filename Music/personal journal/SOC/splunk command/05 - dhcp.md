# Example of a Normal DHCP Log

A common DHCP log entry looks like this:

Mar 06 10:22:31 dhcpd: DHCPDISCOVER from 00:0c:29:78:10:23 via eth0  
Mar 06 10:22:31 dhcpd: DHCPOFFER on 192.168.1.50 to 00:0c:29:78:10:23 via eth0  
Mar 06 10:22:32 dhcpd: DHCPREQUEST for 192.168.1.50 from 00:0c:29:78:10:23 via eth0  
Mar 06 10:22:32 dhcpd: DHCPACK on 192.168.1.50 to 00:0c:29:78:10:23 via eth0

This is generated by a DHCP server using **ISC DHCP**.

---

# Breaking Down the DHCP Log

Let’s analyze one line.

Mar 06 10:22:31 dhcpd: DHCPDISCOVER from 00:0c:29:78:10:23 via eth0

|Field|Meaning|
|---|---|
|Mar 06 10:22:31|Timestamp|
|dhcpd|DHCP server service|
|DHCPDISCOVER|Client searching for DHCP server|
|00:0c:29:78:10:23|Client MAC address|
|eth0|Network interface|

Meaning:

➡️ A device with MAC **00:0c:29:78:10:23** joined the network and is requesting an IP address.

---

# Step-by-Step DHCP Process in Logs

### 1️⃣ Discover

DHCPDISCOVER from 00:0c:29:78:10:23

Client broadcasts to find a DHCP server.

---

### 2️⃣ Offer

DHCPOFFER on 192.168.1.50 to 00:0c:29:78:10:23

Server offers IP **192.168.1.50**.

---

### 3️⃣ Request

DHCPREQUEST for 192.168.1.50 from 00:0c:29:78:10:23

Client accepts the offered IP.

---

### 4️⃣ Acknowledge

DHCPACK on 192.168.1.50 to 00:0c:29:78:10:23

Server confirms the assignment.

This is the **full DHCP process**, defined by **Dynamic Host Configuration Protocol**.

# Another Normal DHCP Log Example

Mar 06 11:03:15 dhcpd: DHCPREQUEST for 192.168.1.55 from c8:60:00:1a:aa:77 via eth0  
Mar 06 11:03:15 dhcpd: DHCPACK on 192.168.1.55 to c8:60:00:1a:aa:77 via eth0

Meaning:

|Item|Value|
|---|---|
|Device MAC|c8:60:00:1a:aa:77|
|Assigned IP|192.168.1.55|
|Interface|eth0|

So the device **successfully obtained an IP address**.

# Common DHCP Log Events

|Event|Meaning|
|---|---|
|DHCPDISCOVER|Device looking for DHCP server|
|DHCPOFFER|Server offering IP|
|DHCPREQUEST|Client requesting offered IP|
|DHCPACK|Server confirms IP|
|DHCPNAK|Server rejects request|
|DHCPRELEASE|Client releases IP|

---

# Suspicious Activity in DHCP Logs

Security analysts watch for:

### Unknown devices

DHCPDISCOVER from unknown MAC

Possible **unauthorized device**.

---

### Too many requests

1000 DHCPDISCOVER messages

Possible **DHCP starvation attack**.

---

### Rogue DHCP server

Multiple DHCPOFFER responses

Could indicate **fake DHCP server performing MITM attack**.

---

# Why DHCP Logs Matter in Security

DHCP logs help track:

| Investigation        | Example                 |
| -------------------- | ----------------------- |
| Identify device      | MAC → IP mapping        |
| Track attacker       | Which device used an IP |
| Detect rogue systems | Unknown MAC addresses   |
| Incident response    | Map network activity    |
# 1️⃣ View All DHCP Logs

Basic search to see DHCP events.

index=network sourcetype=dhcp

This retrieves all DHCP-related logs.

---

# 2️⃣ Show DHCP IP Assignments

Find which **IP addresses were assigned to devices**.

```
index=network sourcetype=dhcp "DHCPACK"  
| table _time host client_ip mac_address
```
Shows:

- Time
    
- DHCP server
    
- Client IP
    
- MAC address
    

---

# 3️⃣ Find All Devices That Requested an IP

```
index=network sourcetype=dhcp "DHCPDISCOVER"  
| stats count by mac_address  
| sort -count
```
This shows **which MAC addresses are requesting IPs the most**.

Useful for detecting:

- suspicious devices
    
- DHCP flooding attacks
    

---

# 4️⃣ Map MAC Address → IP Address

Very useful during investigations.

```
index=network sourcetype=dhcp "DHCPACK"  
| stats values(client_ip) by mac_address
```

This tells you:

MAC address → assigned IP

SOC analysts use this to identify **which device used an IP during an attack**.

---

# 5️⃣ Find New Devices Joining Network

```
index=network sourcetype=dhcp "DHCPDISCOVER"  
| stats earliest(_time) as first_seen by mac_address  
| sort first_seen
```

Shows **when each device first appeared** on the network.

Good for detecting:

- rogue devices
    
- unauthorized laptops
    

---

# 6️⃣ Detect DHCP Flooding Attack

A **DHCP starvation attack** floods the server with requests.

```
index=network sourcetype=dhcp "DHCPDISCOVER"  
| stats count by src_ip  
| where count > 50
```

If a device sends **too many requests**, it might be malicious.

---

# 7️⃣ Find DHCP Failures

Look for rejected requests.

```
index=network sourcetype=dhcp "DHCPNAK"  
| table _time src_ip mac_address
```

These events may indicate:

- misconfiguration
    
- spoofing attempts
    

---

# 8️⃣ Track IP Usage Over Time

```
index=network sourcetype=dhcp "DHCPACK"  
| timechart count by client_ip
```

Shows **IP assignment trends over time**.

---

# 9️⃣ Find Most Active DHCP Clients

```
index=network sourcetype=dhcp  
| stats count by mac_address  
| sort -count  
| head 10
```
Displays **top 10 devices requesting IPs**.

---

# 🔟 SOC Investigation Example

Imagine an attacker used IP:

192.168.1.55

You want to find **which device used it**.

index=network sourcetype=dhcp client_ip=192.168.1.55  
| table _time mac_address host

Now you can identify the **actual device behind that IP**.
