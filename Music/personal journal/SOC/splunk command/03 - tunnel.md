**Tunnel logs** record information about **tunnel creation, traffic flow, and tunnel termination**.

These logs are usually generated by:

- VPN servers
    
- Firewalls
    
- Network gateways
    
- IDS/IPS systems
    
- SIEM platforms like **Splunk Enterprise**
    

They help security analysts monitor:

- Who created the tunnel
    
- From where
    
- When it started
    
- What traffic passed through it

## Common Tunneling Protocols

Some widely used tunneling technologies:

|Protocol|Purpose|
|---|---|
|**SSH Tunneling**|Securely forward traffic through SSH|
|**IPsec**|Secure VPN communication|
|**GRE**|Encapsulate packets for routing|
|**L2TP**|VPN tunneling|
|**DNS Tunneling**|Hide data in DNS queries|
|**ICMP Tunneling**|Send data in ping packets|

Attackers often abuse tunneling protocols to **bypass detection**.

# Typical Tunnel Log Fields

A tunnel log might contain:

|Field|Description|
|---|---|
|Timestamp|When the event occurred|
|Source IP|Where the tunnel originated|
|Destination IP|Tunnel endpoint|
|Protocol|SSH / IPsec / GRE etc|
|Tunnel ID|Unique identifier|
|Bytes sent/received|Traffic volume|
|Session duration|How long tunnel stayed active|
|Username|Authenticated user|
|Status|Success / Failure|
```js 
example:
2026-03-06 12:44:21
TunnelID=9821
User=harsh
SourceIP=192.168.1.20
DestinationIP=10.0.0.5
Protocol=SSH
Action=Tunnel Created
BytesSent=50234
BytesReceived=71231
Status=Success
```

# Suspicious Tunnel Activity (What Analysts Look For)

Security analysts check for:

⚠️ **Unusual destination IPs**  
⚠️ **High data transfer in DNS queries**  
⚠️ **Long-lived tunnels**  
⚠️ **Connections to suspicious countries**  
⚠️ **Repeated tunnel creation attempts**

Example suspicious Splunk query:

index=network_logs action="tunnel-created"  
| stats count by src_ip dest_ip protocol

This shows **which IPs are creating tunnels most frequently**.

# 8. Tunnel Logs Analysis in SIEM (Splunk Example)

Basic analysis steps:

### Step 1 — Find Tunnel Events

```
index=vpn_logs OR index=network_logs  
tunnel
```

---

### Step 2 — Identify Top Tunnel Users

```
index=vpn_logs  
| stats count by user
```

---

### Step 3 — Check Large Transfers

```
index=vpn_logs  
| stats sum(bytes) by src_ip
```

---

### Step 4 — Detect Suspicious Destinations

```
index=vpn_logs  
| stats count by dest_ip  
| sort -count
```

# 9. Real Attack Example

A hacker infects a machine with malware.

The malware creates a **DNS tunnel** like:

data.stolenfiles.attacker-domain.com

Inside the DNS request, the attacker **encodes stolen data**.

Firewall sees **DNS traffic (normal)** but data is actually **exfiltration**.

Tunnel logs help detect this anomaly.
